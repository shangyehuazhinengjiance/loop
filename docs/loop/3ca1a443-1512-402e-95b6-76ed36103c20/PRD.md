# 产品需求文档 (PRD)：群聊界面 UI/UX 优化

## 1. 背景与目标
当前 AI Native Loop 的群聊界面在信息展示和视觉体验上存在一些不足：缺乏当前 Loop 的上下文信息（名称与创建时间）、消息缺少时间戳、Agent 的中间执行过程展示过于零散导致刷屏，以及他人/Agent 消息的气泡背景色（白色）与整体视觉不协调。
本次优化的目标是提升群聊界面的可用性、信息可读性以及整体视觉美感，同时统一时间显示规范（东八区）。

## 2. 用户故事
1. **作为协作用户**，我希望在对话框顶部显眼位置看到当前 Loop 的名称和创建时间，以便快速明确当前所处的上下文。
2. **作为协作用户**，我希望每条消息都能显示发送时间，以便追溯沟通和执行的时间线。
3. **作为协作用户**，我希望 Agent 的中间执行过程（如工具调用、系统日志）能被折叠或整合展示，以便保持聊天界面的清爽，突出核心结果。
4. **作为协作用户**，我希望非本人的消息（其他人类或 Agent）能有更协调的背景色（非纯白），以便更好地区分消息来源并提升视觉体验。
5. **作为国内用户**，我希望所有展示的时间（顶部创建时间、消息时间）均统一为东八区（UTC+8）时间，以便符合日常阅读习惯。

## 3. 验收标准 (Acceptance Criteria)
- **顶部信息栏**：
  - 显眼位置展示 `Loop.title`。
  - 展示 `Loop.createdAt`，格式化为 `YYYY-MM-DD HH:mm:ss`，强制使用东八区（UTC+8）时区。
- **消息时间戳**：
  - 每条聊天消息气泡旁（或内部底部）展示时间戳。
  - 时间取自 `message.metadata.timestamp` 或 `created_at`，格式化为 `HH:mm` 或 `MM-DD HH:mm`，强制使用东八区（UTC+8）时区。
- **中间过程优化**：
  - 针对 `sender.type === 'system'` 或 `content.type === 'artifact' / 'tool_use'` 的中间态消息，采用折叠面板（Accordion）或紧凑型日志流（Log Stream）进行视觉降噪。
  - 默认收起冗长的执行过程，仅展示当前状态（如“正在读取文件...”），执行完毕后保留一个可展开的执行摘要。
- **气泡背景色调整**：
  - 当前用户（Self）发送的消息保持现有主题色。
  - 其他来源（Agent、其他 Human）发送的消息背景色从纯白色调整为与背景更协调的浅色调（如浅灰 `#F3F4F6` 或主题次要色），确保文字对比度达标且视觉柔和。
