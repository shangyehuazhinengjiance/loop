# PRD：聊天界面 UI 优化与系统头像功能

## 1. 背景与目标
当前系统中的对话界面在可读性上存在优化空间。为了提升用户在 Loop 系统中的沟通体验，降低阅读负担，本次迭代计划参考微信（WeChat）经典的聊天界面 UI 进行重构。同时，由于当前系统缺乏头像功能，为了配合聊天界面的优化并提升身份辨识度，本次迭代将引入系统级别的头像支持，并为未设置头像的用户和 AI 提供默认头像。

## 2. 用户故事
*   **作为一名用户**，我希望系统能为我提供一个默认头像，以便在聊天界面中清晰地展示我的身份，即使我没有主动上传头像。
*   **作为一名用户**，我希望我发送的消息显示在右侧，AI 或其他成员的消息显示在左侧，以便我能一眼区分消息的发送方。
*   **作为一名用户**，我希望消息被包裹在具有不同背景色的气泡中（例如：我的消息是绿色，他人是白色/浅灰色），以便在长篇对话中快速定位。
*   **作为一名用户**，我希望消息之间有合理的间距，并在必要时显示时间戳，以便我能清晰地追踪对话的时间线。

## 3. 功能需求与验收标准

### 3.1 系统头像与默认头像（新增）
*   **默认头像机制**：当用户或 AI 实体没有配置自定义头像 URL 时，前端需展示默认头像。
*   **默认头像样式**：可采用统一的默认用户轮廓图标（SVG/PNG），或基于用户昵称/AI名称的首字母生成带有固定背景色的文字头像。
*   **验收标准**：所有参与对话的用户和 AI 在聊天界面中均有头像显示；默认头像加载迅速，且在不同分辨率下保持清晰（推荐使用 SVG 或 CSS 绘制）。

### 3.2 消息布局与对齐
*   **当前用户（Self）**：头像位于右侧，消息气泡靠右对齐。
*   **其他用户/AI（Others）**：头像位于左侧，消息气泡靠左对齐。
*   **验收标准**：无论屏幕宽度如何变化，左右对齐逻辑保持稳定，且气泡最大宽度不超过屏幕/容器宽度的 80%（避免单行过长影响阅读）。

### 3.3 气泡样式与颜色
*   **背景色**：当前用户发送的消息气泡背景色参考微信绿（如 `#95ec69`），其他用户/AI 的消息气泡背景色为纯白（`#ffffff`）或浅灰，并带有轻微的阴影或边框以区分背景。
*   **圆角与边距**：气泡需设置合适的圆角（如 `8px`），内部文字与气泡边缘保持舒适的内边距（Padding）。
*   **验收标准**：颜色对比度符合无障碍阅读标准，气泡样式在亮色/暗色模式下均表现良好。

### 3.4 细节展示（头像、昵称与时间戳）
*   **头像展示**：统一尺寸（如 `40x40px`），方形圆角或圆形。
*   **昵称**：在群聊或多 Agent 场景下，左侧消息气泡上方需显示发送者昵称（字号较小，颜色较浅）。
*   **时间戳**：两条消息间隔超过一定时间（如 5 分钟），则在中间居中显示灰色的时间戳。
*   **验收标准**：头像与消息气泡顶部对齐良好；时间戳格式清晰（如“昨天 14:30”或“14:30”）。

### 3.5 富文本与 Markdown 兼容
*   **内容渲染**：AI 输出的 Markdown 格式（加粗、列表、代码块等）必须在气泡内正确渲染。
*   **验收标准**：代码块需支持横向滚动，长文本或长链接需支持自动换行，绝不能撑破气泡或导致页面横向滚动。
